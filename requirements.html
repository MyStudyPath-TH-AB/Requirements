Zustimmung Datennutzung

Zustimmung-Datennutzung

Anforderung

Nutzer müssen der Nutzung ihrer Daten zustimmen, bevor sie die Anwendung verwenden können.

Anmerkungen

  • Muss bei erster Anmeldung erfolgen (bzw bis Zustimmung gegeben wurde).
  • Zustimmung sollte jederzeit widerrufbar sein.

Frontend Beratung

Übersicht

Übersicht

Anforderung

Es muss eine Übersicht der Studenten angezeigt werden können.

Anmerkungen

  • Wenn man auf einen Studenten klickt, soll die Detailansicht des Studenten angezeigt werden.
  • Die Studentenliste soll farblich sortiert sein.
  • Nach dem login soll man hier landen.

Detailansicht

Detailansicht

Anforderung

Es muss eine Detailansicht eines bestimmten Studenten angezeigt werden können.

Anmerkungen

  • Die Detailansicht soll alle relevanten Informationen über den Studenten enthalten.
  • Es soll möglich sein, Kommentare zu dem Studenten hinzuzufügen.
  • Es soll einen Zurück Button geben, um zur Übersicht zurückzukehren.

Authentifizierung

Logout

Anforderung

Das System muss sich aus dem Hochschulsystem ausloggen können.

Anmerkungen

  • Es muss sichergestellt werden, dass die Session des Nutzers im Hochschulsystem auch wirklich geschlossen wird, damit keine unbefugten Zugriffe möglich sind.
  • vgl. Logout Frontend

Timeout

Anforderung

Bei zu langer Inaktivität soll der Nutzer automatisch ausgeloggt werden.

Anmerkungen

Auch relevant, wenn der Nutzer den Browser oder die Seite schließt, damit keine unbefugten Zugriffe möglich sind.

Fachunterscheidung

Anforderung

Fächer innerhalb des Studienplans sollen sichtbar voneinander unterschieden werden können.

Anmerkungen

  • Abgrenzungen anhand Fakultäten, Studiengängen oder Wahl/Pflicht.
  • Semester müssen klar voneinander abgegrenzt sein.
  • Fächer sollen klickbar sein, um weitere Informationen zu erhalten (siehe Fachübersicht).

Empfehlungen

Anforderung

Dem Benutzer müssen Empfehlungen angezeigt werden.

Anmerkungen

  • Diese sollen strukturiert angezeigt werden. z.B. Empfehlungen hinsichtlich Schwerpunktewahl, Wahl der Wahlfächer, Prüfungsterminkollisionen, etc.
  • Anzeige SPO + APO?

Termine-Fristen

Anforderung

Fortschrittsanzeige

Anforderung

Es wird eine Fortschrittsanzeige für den Studienverlaufsplan angezeigt.

Anmerkungen

  • Meilensteine könnten berücksichtigt werden.
  • Eine Anzeige der erreichten und benötigten ECTS Punkte ist erwünscht.

Anmerkungen

Validierung

Anforderung

Eingaben des Nutzers müssen validiert werden.

Anmerkungen

  • Kein Problem bei Dropdown Feldern, aber bei Freitext Feldern muss sichergestellt werden, dass die Eingaben sinnvoll/nicht schädlich sind.

Eingabenansicht

Anforderung

Der Nutzer muss die Möglichkeit haben, zusätzliche persönliche Informationen einzugeben.

Anmerkungen

  • Informationen könnten z.B. vorrangehende Ausbildungen, die momentane Semesterzahl, oder so empfundene Problemfächer sein.
